# 嵌入式代码风格

## 使用流程

1. 先读取当前上下文提供的风格说明；若存在 `代码风格.md`，将其作为最高优先级。
2. 再观察当前文件和同模块代码，判断哪些写法符合本规范，哪些写法需要在本次修改范围内调整。
3. 编写或修改代码时按本 skill 的规范执行，同时保持接口和行为稳定。
4. 若用户明确要求行为修改，再同时处理语义变化。
5. 完成后按 `references/风格检查.md` 自查。

## 引用文件

- 写 C/C++ 代码前，读取 `references/代码风格.md`。
- 涉及命名、namespace、函数排序、class 布局时，读取 `references/命名与结构.md`。
- 需要新增或调整函数注释时，读取 `references/注释与可读性.md`。
- 完成改动前，读取 `references/风格检查.md`。

## 优先级

1. 用户当前明确要求。
2. 当前上下文提供的 `代码风格.md`。
3. 本 skill。
4. 当前文件中已经符合本规范的写法。

## 底线

- 风格调整不能顺手改硬件引脚、协议字节、大小端、定时参数、任务优先级、控制阈值、执行顺序。
- 公共接口重命名要评估调用点和兼容性；不要为了单纯风格造成额外迁移风险。
- 大范围格式化必须是用户明确要求或任务本身需要。
