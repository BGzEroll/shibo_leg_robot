# 命名与结构

## namespace 与 .cpp 定义

尽可能只在 `.h` 中使用 namespace 包住导出声明；`.cpp` 中用 `::` 定义 namespace 成员。

头文件：

```cpp
#ifndef CONTROLLER_H
#define CONTROLLER_H

#include <Arduino.h>

namespace controller
{
    bool middle_calibration_success();
    void mark_middle_calibration_success();
    bool request_middle_calibration();
    void update(uint32_t tick_ms);
    void init();
    void task_entry(void *arg);
}

#endif
```

源文件：

```cpp
#include "controller.h"

#include "actions.h"
#include "balance_core.h"

static uint32_t update_count = 0;

/**
 * @brief 更新内部统计计数
 */
static void update_counter()
{
    update_count++;
}

/**
 * @brief 查询舵机中位校准流程是否已经成功执行
 *
 * @return 已成功执行时返回 true
 */
bool controller::middle_calibration_success()
{
    return false;
}

/**
 * @brief 执行一次上层控制器更新
 *
 * @param tick_ms 本次更新周期，单位毫秒
 */
void controller::update(uint32_t tick_ms)
{
    update_counter();
}
```

`.cpp` 中不要使用 `namespace action = controller::actions;` 这类 namespace 别名。需要访问命名空间成员时，写完整限定名。

`.cpp` 中也不要使用 `using namespace ...;` 或 `using controller::mode_id;` 这类 `using` 简写。公共库、框架、SDK 源码或示例本身已有的 `using` 不必为了本规范修改；自己写的项目代码优先写完整限定名。

推荐：

```cpp
controller::actions::begin_mode(state, controller::mode_id::BALANCE);
```

避免：

```cpp
namespace action = controller::actions;
action::begin_mode(state, controller::mode_id::BALANCE);

using controller::mode_id;
mode_id mode = mode_id::BALANCE;
```

`.cpp` 内可以写局部 forward declaration，用来隐藏只允许当前编译单元或固定聚合模块调用的内部入口。不要把这种声明复制到多个无关文件；如果函数已经是公共契约，应放到对应头文件。

允许：

```cpp
namespace host_comm
{
    void init();
}
```

## 头文件函数排序

namespace 头文件中的函数声明按下面顺序排列：

1. 对外开放 API。
2. `init()`。
3. RTOS 任务入口，例如 `task_entry(void *arg)`、`core_task_entry(void *arg)`。

对应 `.cpp` 中：

- 内部 `static` 状态和 `static` helper 放在对外 API 前面。
- 对外 API 放在文件后段。
- 对外 API 顺序尽量与 `.h` 声明一致。

推荐：

```cpp
namespace host_comm
{
    QueueHandle_t remote_queue();
    bool vision_latest(vision_measurement &out);
    void init();
    void task_entry(void *arg);
}
```

## class 组织

class 使用 `snake_case` 命名。

使用 `public` / `private` 分组表达意图：

1. 构造、析构等生命周期函数。
2. `init()` 或初始化相关函数。
3. 对外 API。
4. 内部 helper。
5. 数据成员。

可以重复使用 `public:` / `private:` 做分组。不要把构造函数、API、helper、成员变量混在一段里。

推荐：

```cpp
class pwm_servo
{
    public:
        pwm_servo(uint8_t pin, uint8_t channel, uint32_t freq, uint8_t resolution);

    public:
        void init();

    public:
        void set_angle(uint16_t angle);

    private:
        void set_us(uint16_t us);

    private:
        uint8_t pin;
        uint8_t channel;
        uint32_t freq;
        uint8_t resolution;
};
```

## 结构体与枚举

- `struct` 名使用 `snake_case`，不加 `_t`。
- `enum class` 名使用 `snake_case`，枚举值使用 `UPPER_SNAKE_CASE`。
- 结构体成员使用 `snake_case`。
- 运行时状态结构体放在真正拥有和修改它的模块附近。

推荐：

```cpp
enum class mode_id : uint8_t
{
    BOOT = 0,
    BALANCE,
    STOP
};

struct control_input
{
    uint16_t raw_buttons = 0;
    uint16_t buttons = 0;
    float linear_cmd = 0.0f;
};
```

## 公共契约

- 头文件只放公共契约。
- 私有状态、接收缓冲、定时器、临时 helper 放 `.cpp`，用 `static` 限制作用域。
- 不创建“所有模块共用类型”的杂物头文件，除非这个类型确实跨模块稳定共享。
