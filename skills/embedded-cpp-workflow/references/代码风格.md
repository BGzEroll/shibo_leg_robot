# 代码风格

## 命名

- 统一使用 `snake_case`，包括文件名、命名空间、类名、结构体名、枚举名、函数名、变量名和成员名。
- 结构体类型、枚举类型、普通类型不要加 `_t` 后缀。
- 常量使用 `UPPER_SNAKE_CASE`，例如 `VISION_TIMEOUT_MS`、`FRAME_HEADER`。
- 布尔变量使用可直接读出判断含义的名字，例如 `connected`、`valid`、`ready`。
- 复用项目已有领域词汇，不创造同义新词。
- 本规范约束自己编写或维护的项目代码；公共库、框架、SDK 已有 API、类型名、宏名、回调签名和第三方头文件不需要为了本规范改名。

推荐：

```cpp
static constexpr uint32_t VISION_TIMEOUT_MS = 350;

struct status_snapshot
{
    uint32_t timestamp_us = 0;
    int16_t dx = 0;
    int16_t dy = 0;
    bool valid = false;
};

enum class mode_id : uint8_t
{
    BOOT = 0,
    BALANCE,
    STOP
};
```

避免：

```cpp
struct status_snapshot_t
{
    int dx;
    short dy;
};
```

公共库边界可以保留原名：

```cpp
void set_serial(HardwareSerial *serial);
void task_entry(void *arg);
```

## 类型

- 整数类型必须优先使用显式定宽类型，例如 `int16_t`、`uint8_t`、`uint32_t`。
- 避免使用 `int`、`short`、`long` 这类宽度不明确的类型。
- 与库 API 交互时，如果库要求 `int` 等类型，可以在调用边界使用，并避免把这种类型扩散到项目数据结构。
- 协议字段、数组下标、计时值、长度值要按含义选择类型，不为了省事统一写成同一种整数。

推荐：

```cpp
static uint8_t rx_buf[256];
static uint32_t rx_len = 0;

static void parse_axis(const uint8_t *payload)
{
    int16_t raw = (int16_t)(payload[0] | (payload[1] << 8));
    float axis = (float)raw * 1.0e-3f;
}
```

## 缩进与括号

- 默认使用 4 空格缩进，不使用 tab。
- 函数、`namespace`、`class`、`struct`、`enum` 的左括号独占一行。
- `if`、`for`、`while`、`switch` 的左括号也换行。
- 每进入一层作用域都必须增加一级缩进。`namespace` 内写 `struct`，`struct` 内写成员，都必须体现层级。
- 较长函数声明第一行放不下时可以换行；续行相对首行增加 4 空格缩进，不使用 tab，也不为了对齐括号填充大量空格。

推荐：

```cpp
namespace host_comm
{
    struct remote_data
    {
        uint32_t timestamp_us = 0;
        uint16_t buttons = 0;
        float axes[6]{};
    };

    bool remote_latest(remote_data &out);
    bool recover_ready(action_state &state,
        const balance_core::status_snapshot &status,
        uint32_t tick_ms,
        float pitch_limit,
        float rate_limit,
        uint32_t hold_ms,
        uint32_t timeout_ms);
    void init();
    void task_entry(void *arg);
}
```

避免：

```cpp
namespace host_comm
{
struct remote_data
{
uint32_t timestamp_us = 0;
};
}
```

## 初始化与列表

单行初始化允许写成一行：

```cpp
static const uint8_t FRAME_HEADER[] = {0xFF, 0xAA};
static constexpr uint32_t VISION_TIMEOUT_MS = 350;
```

多行数组、结构体、列表初始化如果使用 `=`，则 `=` 后换行再写 `{`：

```cpp
static const uint8_t FRAME_HEADER[] =
{
    0xFF,
    0xAA,
    0x02
};

motor_config config =
{
    .left_pin = 32,
    .right_pin = 33,
    .pwm_freq = 20000
};
```

避免 `= {` 后再换行：

```cpp
static const uint8_t FRAME_HEADER[] = {
    0xFF,
    0xAA
};
```

构造函数初始化列表按 C++ 常见格式换行即可，不要求套用上面的 `=` 规则：

```cpp
pwm_servo::pwm_servo(uint8_t pin, uint8_t channel)
    : pin(pin), channel(channel)
{
}
```

## 表达式格式

- 运算符两侧保留空格，尤其是位运算和移位：写 `1 << 2`，不要写 `1<<2`。
- 复合表达式可以拆成有物理意义或协议意义的中间变量。
- 短 guard 可以写成单行，但表达式内部仍要有空格；复杂条件要展开。
- 函数内未使用的参数不要写 `(void)arg;` 这类占位语句；保留参数名即可。
- 不为了列对齐制造大面积 diff。

推荐：

```cpp
uint16_t buttons = (uint16_t)(payload[0] | (payload[1] << 8));

if(!rx_len){return;}

if((buttons & BTN_SELECT) && (pressed_buttons & BTN_X))
{
    begin_mode(mode_id::KICK_PLACE);
}

/**
 * @brief RTOS 任务入口
 *
 * @param arg RTOS 任务参数
 */
void host_comm::task_entry(void *arg)
{
    while(true)
    {
        update_rx();
        vTaskDelay(1);
    }
}
```

避免：

```cpp
uint16_t buttons=(uint16_t)(payload[0]|(payload[1]<<8));

void host_comm::task_entry(void *arg)
{
    (void)arg;
    while(true)
    {
        update_rx();
        vTaskDelay(1);
    }
}
```

## include

`.cpp` 文件 include 顺序：

1. 第一个 include 必须是同名头文件。
2. 空一行。
3. include 项目内依赖。
4. 必要时再 include 框架、标准库或第三方库。

推荐：

```cpp
#include "controller.h"

#include "actions.h"
#include "balance_core.h"
#include "host_comm.h"
#include "freertos/task.h"
```

头文件只包含必要 include；能通过前置声明解决且不牺牲清晰度时，优先减少头文件依赖。
